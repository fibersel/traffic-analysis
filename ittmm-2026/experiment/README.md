# Экспериментальное исследование алгоритма активного управления очередями RED

Данный проект содержит материалы для проведения эксперимента по исследованию динамики очередей при использовании алгоритма активного управления перегрузками **RED (Random Early Detection)**. Эксперимент проведен в среде имитационного моделирования **NS-2**, что соответствует методологии диссертационных исследований РУДН (Королькова А.В., 2010; Велиева Т.Р., 2019).

##  Описание

Цель работы — верификация математической модели системы «TCP-источник — RED-маршрутизатор» и анализ устойчивости системы при различных параметрах алгоритма (пороги $q_{min}, q_{max}$, вес сглаживания $w_q$).

**Почему NS-2 вместо Kathara?**
В ходе подготовки эксперимента было выявлено, что контейнерная среда эмуляции сети (Kathara/Docker) имеет ограничения ядра Linux, не позволяющие загружать модуль дисциплины очередей `sch_red`. В результате RED не активируется, происходит откат на `fq_codel` или `noqueue`. Для получения достоверных данных об алгоритме RED используется эталонный симулятор **NS-2**, который имеет встроенную программную реализацию алгоритма.

##  Требования

Для запуска эксперимента необходима операционная система **Linux** (рекомендуется Ubuntu 24.04 LTS) со следующим ПО:

1.  **NS-2 (Network Simulator 2)** — для имитации сети.
2.  **Python 3** — для обработки результатов трассировки.
3.  **LaTeX** — для компиляции статьи (опционально).
4.  **Библиотеки Python:** `matplotlib`, `numpy`, `pandas`.

##  Установка зависимостей

### 1. Установка NS-2
В Ubuntu 24.04 NS-2 может отсутствовать в стандартных репозиториях. Попробуйте установить через apt:
```bash
sudo apt update
sudo apt install ns2 nam xgraph
```
*Проверка установки:*
```bash
ns
# Должно появиться приглашение %, введите exit для выхода
exit
```

### 2. Установка Python библиотек
```bash
pip3 install matplotlib numpy pandas
```

### 3. Установка LaTeX (для статьи)
```bash
sudo apt install texlive-full texlive-science texlive-lang-cyrillic
```

##  Запуск эксперимента

Все шаги автоматизированы скриптом `run_simulation.sh`, но их можно выполнить вручную.

### Шаг 1: Запуск имитации NS-2
Скрипт создаёт топологию сети, настраивает RED и генерирует трафик.
```bash
chmod +x run_simulation.sh
./run_simulation.sh
```
*Или вручную:*
```bash
ns red_simulation.tcl
```
**Результат:** Создаются файлы трассировки `red_queue.tr`, `queue.tr`, `cwnd.tr`.

### Шаг 2: Обработка данных
Скрипт на Python извлекает данные о длине очереди из trace-файла и строит графики.
```bash
python3 process_queue.py
```
**Результат:** Создаются файлы `queue_graph.pdf` и `queue_graph.png`.

### Шаг 3: Компиляция статьи (опционально)
Если вы готовите статью для конференции/журнала.
```bash
pdflatex main.tex
pdflatex main.tex  # Запустить дважды для корректных ссылок
```
